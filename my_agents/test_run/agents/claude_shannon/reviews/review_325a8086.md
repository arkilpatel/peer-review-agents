# Review: Instant Gaussian Stream: Fast and Generalizable Streaming of Dynamic Scene Reconstruction via Gaussian Splatting

### Summary

This paper (Yan et al., Peking University / Pengcheng Laboratory, ICLR 2025 — likely CVPR 2025 track) proposes Instant Gaussian Stream (IGS), a framework for streaming dynamic scene reconstruction using 3D Gaussian Splatting. The key contributions are: (1) a generalized Anchor-driven Gaussian Motion Network that projects multi-view 2D motion features into 3D space, driving Gaussian motion via anchor points; (2) a Key-frame-guided Streaming Strategy to handle temporally complex scenes while mitigating error accumulation. IGS achieves 2.67s per-frame reconstruction with PSNR of 34.15 dB and 204 FPS rendering — a significant improvement over 3DGStream (CVPR 2024, ~10s+ per-frame) and other streaming methods. In-domain and cross-domain evaluations demonstrate generalization. Overall assessment: a solid, well-executed engineering contribution to dynamic scene reconstruction with clear practical value for real-time streaming applications.

### Novelty Assessment

**Verdict: Moderate**

3D Gaussian Splatting (Kerbl et al., 2023) for dynamic scenes has been addressed by multiple works: 3DGStream (CVPR 2024), Spacetime-GS (CVPR 2024), Kplanes, StreamRF (CVPR 2023). The anchor-based motion network and key-frame guided strategy are incremental but practically important innovations. The cross-domain generalization (training on one scene, testing on another) is a notable advance over prior per-scene optimization approaches, which is genuinely novel for the streaming Gaussian paradigm.

---

### Technical Soundness

The anchor-driven motion network uses multi-view 2D motion features projected to 3D — this is architecturally sound and avoids per-frame optimization. The key-frame guidance provides a regularization mechanism against error accumulation in streaming, which is a real and known problem in video-based Gaussian methods. The claimed 2.67s per-frame versus 10s+ for prior streaming methods (3DGStream) represents a ~4× speedup. The 204 FPS rendering rate confirms real-time capability. The architecture choice to use a generalized network (one network for all frames, not per-frame optimization) is the core technical innovation that enables both speed and cross-domain generalization.
