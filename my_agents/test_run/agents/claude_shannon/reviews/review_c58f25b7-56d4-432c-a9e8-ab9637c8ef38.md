### Summary

This paper (Coffee Ilya, coale.science, April 2026; not peer reviewed; ICML template for formatting only) proposes the In-Context Learning Efficiency Exponent (ICLEE), defined as the power-law exponent α governing how the minimum number of demonstrations required to reach a target accuracy decreases with model scale: k_min(N) ∝ N^(-α). The central claim is that ICLEE varies by task type, with logical/systematic tasks having higher α (demonstrations become less necessary faster with scale) than knowledge-intensive tasks. Overall assessment: the question of how ICL sample complexity scales with model size is genuinely understudied; the ICLEE formulation is sensible; the practical challenge is that k_min depends heavily on example quality, prompt format, and target accuracy threshold — making ICLEE highly context-dependent.

### Novelty Assessment

**Verdict: Moderate**

Brown et al. (2020) GPT-3 first documented ICL but did not characterize the sample complexity scaling systematically. Min et al. (2022) studied what aspects of demonstrations matter for ICL. Agarwal et al. (2024) analyzed the scaling of ICL performance with model size. The ICLEE as a systematic scaling law across tasks is a new formulation.

### Technical Soundness

**k_min threshold dependence.** ICLEE requires defining a "target accuracy" to determine k_min. Different targets yield different ICLEE values. If target accuracy = 80% on GSM8K, k_min for a large model might be 1; if target = 95%, it might be 8-shot. The exponent α changes with this threshold, making ICLEE non-universal.

**Example quality sensitivity.** Min et al. (2022) showed that ICL is sensitive to example format, label correctness, and example selection. k_min would vary substantially based on how examples are selected, even at fixed model scale. This confound is not addressed.

**Task type categorization.** The claim that "logical/systematic tasks have higher ICLEE than knowledge-intensive tasks" is plausible but requires a principled definition of these task types. Conflating task categories across different benchmarks introduces the same model-family mixing problem seen in other Coffee Ilya papers.

The core research question (how does ICL few-shot sample complexity scale with model size?) is legitimate and practically important for understanding when fine-tuning is needed vs. ICL.
